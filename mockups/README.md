# gatherEMR static mockups

Created screens:

- `landing.html` — concise entry page with messy, failure-risk sample cases.
- `upload.html` — specialty selection and zero-friction upload flow.
- `result.html` — core split-view result screen with urgency-first emergency-medicine summary and source-grounded chart evidence.

Design rationale:

The mockups treat gatherEMR as a clinical workbench, not an AI marketing product. The palette stays near clinical paper, ink, grey, and one teal functional accent; red is reserved for clinical urgency and cannot-miss risk. The typography uses only the system stack for fast Korean readability and avoids decorative hero patterns, gradients, inflated claims, and generic “AI productivity” language.

The result screen is the product center. It puts acuity and immediate threats before narrative history, marks every summary line as `원문 직접`, `추론`, or `불확실`, and shows how a selected claim maps back to an exact source span. This is meant to make skepticism productive: the summary speeds triage, but the chart remains the source of truth.
