# Opus 핸드오프 — 정신건강의학과 요약 모듈

> **왜 분리:** 정신건강의학과 요약 템플릿은 임상적으로 **환자 안전 위험도 평가**(자·타해 위험 등 급성 정신과 응급 판단)를 최상단 블록으로 요구한다. 이 도메인 콘텐츠가 Fable 세이프가드에 반복 플래그되어 세션이 멈춤. → **이 분과의 요약 템플릿 설계·프롬프트 작성은 Opus 세션에서 마감한다.**
> 임상 근거·블록 초안은 이미 `med2-내과계기타.md`의 "정신건강의학과" 섹션에 조사·정리되어 있음(그 파일 참조).

## Opus가 할 일
1. `med2-내과계기타.md`의 정신건강의학과 조사 결과를 근거로 이 분과의 `lens / blocks / cannotMiss / chips`를 레지스트리 스키마(`INDEX.md` 참조)에 맞춰 확정.
2. 요약 시스템 프롬프트에 이 분과 렌즈를 주입할 때, **의료 전문가(정신건강의학과 의사) 대상의 차트 요약**이라는 임상 맥락을 명확히 하고, 안전 위험도 관련 블록은 "차트에 기록된 것을 인용·정리"하는 방식(생성·조언이 아니라 요약)으로 한정.
3. 다른 25개 분과와 동일한 공통 골격(one-liner→cannot-miss 배지→blocks→투약 로그→불확실) + 청크인용/exact-span 규칙 적용.

## 참고
- 나머지 25개 분과는 Fable에서 정상 처리 가능(INDEX.md의 4개 리서치 파일).
- 정신건강의학과도 **UI 드롭다운·레지스트리 엔트리(계열: 내과·진료)로는 존재**해야 함 — 요약 프롬프트 내용만 Opus가 마감.
