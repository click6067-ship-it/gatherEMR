## 5개 클러스터 전체를 관통한 3대 공통 패턴 (설계 정본)
전 34개 조사 단위에서 반복 확인됨:
1. **추이(trend) > 스냅샷.** EF·BNP·Cr/GFR·HbA1c·CBC·DAS28·FIM·성장곡선·객담전환… 전 분과가 "지금 값"이 아니라 "방향과 속도"로 판단. → 요약 항목은 가능하면 `값(시각1)→값(시각2)` 형태.
2. **Cannot-miss는 최상단 경고 배지.** 급성 악화 또는 감별 실패(감염 vs flare, AKI vs CKD, 패혈증, herniation, 기도폐색, 요로패혈증, 잠복기 불일치…)로 수렴. → 스크롤 없이 즉시 보이는 **고정 상단 경고 블록**.
3. **투약/치료 변경 로그는 분과 무관 공통 컴포넌트.** 항응고제·스테로이드·면역억제제·항생제·항암 — "무엇을 왜 바꿨고 반응은?"이 다음 결정의 근거.

## 분과별 요약 = 공통 골격 + 분과 렌즈 (레지스트리 스키마)
각 분과 = `{ id, 한글명, 계열(4그룹), lens[], blocks[], cannotMiss[], chips[] }`
- **공통 골격 (모든 분과 요약에 주입):** one-liner → ⚠️cannot-miss 배지 → (분과 blocks) → 투약 변경 로그 → 불확실/누락. 전부 청크인용 + explicit/derived/uncertain 라벨 + exact-span 클릭.
- **분과 렌즈:** 각 파일의 "먼저 보는 것"=lens, "요약 블록 구성"=blocks(순서), "Cannot-miss"=cannotMiss, "포커스 칩"=chips.
- **포커스 칩 2축 표준화(외과계 리서치 발견):** 위험도 축(출혈/감염/마취/특이위험) + 이력 축(이전 수술·방사선·영상/검사 추이). 이 2축으로 26분과 칩을 일관 설계.
- **진단·지원계 특이(diag 리서치):** "직전 대비(prior comparison)" 블록을 공통 컴포넌트로 — 의뢰 사유/임상질문의 구체성이 품질을 좌우.

## 구현 함의
- 26분과 **전부 동작**(응급의학만 하드코딩 금지). 균등하게 lens 기반으로 작동.
- 드롭다운 = 4계열 optgroup(내과·진료 / 외과·수술 / 진단·지원 / 응급·사회의학) + 26항목.
- 요약 프롬프트 = 공통 시스템 프롬프트 + 선택 분과의 lens/blocks/cannotMiss 주입.
