# motionsites.ai 기법 벤치마킹 — 우리가 가져올 것 (2026-07-04)

**방법:** 테마(의료)가 아니라 **인터랙션 문법**을 뽑는다. motionsites 대부분은 (a)다크 (b)추상 장식배경 (c)브랜드만 갈면 재사용 = 우리가 피하는 슬롭. → **기법(reveal·follow·parallax·assemble)은 훔치되 "스킨"은 버리고, 각 기법을 제품 의미(비식별·판독·근거)에 묶는다.** 이게 반슬롭 레퍼런스-first의 정석.

**메타 관찰:** motionsites `/backgrounds`(오로라·플라즈마·네온·실크 수백개) = 전부 공유 다운로드 에셋 = 남들과 동일 = **살 이유 0.** 가치는 홈의 *인터랙티브 히어로 기법*에만 있음.

---

## ★ KEEP — 가져올 기법 (우선순위)

### 1. Reveal Hero → **"비식별 리빌"** (당신이 콕 집은 것 · 최우선)
- **원본:** 얼굴 위 마우스 호버 → 마스크가 씌워짐 (mask 광고).
- **우리 번역:** 히어로/데모에서 **차트의 식별자 위에 호버 → ███ 마스크가 슬라이드로 덮임** (또는 역: 호버로 "무엇이 가려졌나" 잠깐 보임). 
- **왜 강력한가:** 우리 핵심 가치("PHI를 가린다")를 *설명*이 아니라 *손으로 만지게* 함. 삭제한 S4(신뢰 섹션)를 이 마이크로 인터랙션 하나가 대체. 제품명 바꾸면 못 쓰는 = 반슬롭 통과.

### 2. Vision Reveal (텍스트 마스크 리빌) → **"잉크 와이프 헤드라인"**
- **원본:** 로드/스크롤 시 큰 타이포가 클립마스크로 그려짐.
- **우리 번역:** H1 "차트는 길다. 봐야 할 것은 짧다."가 **잉크가 번지듯 clip-path 와이프**로 등장. 우리 잉크 모티프와 일치. (라이브러리 0, CSS만.)

### 3. Cursor Follow → **"판독 렌즈"**
- **원본:** 빛/오브가 커서를 따라옴.
- **우리 번역:** 히어로 차트 위를 **부드러운 하이라이트 밴드가 커서 따라 이동**하며 지나는 줄을 살짝 밝힘 = "AI가 줄 단위로 읽는다"의 시각화. 라이트·절제.

### 4. Layered Depth (패럴랙스) → **"차트 심도"**
- **원본:** 레이어별 다른 속도 = 3D 없이 깊이.
- **우리 번역:** 히어로의 차트텍스트/요약/ECG 3레이어가 **미세하게 다른 패럴랙스** → 종이 위 깊이감. (이미 레이어 구조 있음, 속도차만.)

### 5. Data-viz 스파크라인 → **"정직한 Troponin 추세"** (이전 Fable v2 아이디어 계승)
- **원본:** Transform Data/Dashboard류의 실데이터 모션.
- **우리 번역:** S2 데모에 **Troponin 0.8→2.1 미니 스파크라인**, 마지막 21:10은 **속 빈 점("pending")**. 진짜 데이터 = 제품 시연형(리서치 결론과 일치).

### 6. Scroll-assemble (Scroll Landing) → **"요약 조립"** (이미 부분 구현)
- S2에서 스크롤 진입 시 요약 문장→원문 **연결선이 그려짐**. 유지·강화.

- **Loader → "트리아지 로더":** 진입 시 ECG 1스트로크 그려지고 gatherEMR resolve. **LCP 해치면 폐기** (의료는 로딩=신뢰). 옵션.

## 결론 — 우리 랜딩에 반영할 3가지 (구현 우선순위)
1. **비식별 리빌**(Reveal Hero) — 히어로 또는 독립 마이크로 섹션. *최우선, 가장 온-테마.*
2. **잉크 와이프 H1**(Vision Reveal) — 즉시, 저비용.
3. **판독 렌즈**(Cursor Follow) + **Troponin 스파크라인**(data-viz) — 데모 강화.
