# 랜딩 개편 — "The Weld Scan Journey" (2026-07-04 확정 설계)

> 히어로를 **스크롤 스크럽 시네마틱**으로 전환하고, 다크→라이트로 해소되며 스크럽 끝에
> 정제된 유리 대시보드가 재조립되는 랜딩. 나머지 섹션은 라이트 유지 + "레이저 스캔라인"
> 모티프로 관통. 근거: 확정 스틸(`docs/research/hero-video/confirmed-first-frame.png`) +
> SynapseX 스크럽 레시피(scroll→currentTime, `!video.seeking` 가드).

## 1. 목표 / 배경
- 현 랜딩(v4.2)은 **웜 라이트** 계열. 새 Flip Scan 영상은 **다크·블루·시네마틱** → 톤 충돌.
- 사용자 결정으로 **스크롤 스크럽 여정**(다크→라이트 해소)을 채택 → 제품 은유(raw→refined,
  어둠→빛)와 스크럽이 맞물림.
- 스크럽 채택의 부수효과: **심리스 루프 불필요**(스크럽은 영상을 앞뒤로 감음). 영상 딜리버러블이
  루프 → **선형 1방향**으로 단순화.

## 2. 확정 결정 로그 (사용자 승인)
1. **전체 톤 = 스크럽 여정.** 세로로 긴 핀 고정 히어로, 스크롤=영상 프레임. 스크럽 완료 시
   화면이 밝아지며 라이트 제품 섹션으로 해소.
2. **스크럽 페이오프 = 유리 대시보드 재조립.** 변환 완료 순간, 기존 Bio-Age 대시보드
   (12문장 카운트업 주인공 카드 + 보조 카드 4장)가 유리 카드로 조립되며 등장. "raw→refined =
   이 대시보드."
3. **폰트·라이트 팔레트 유지.** Space Mono 미채택(= SynapseX 스킨 차용 금지). 기존 sans/mono +
   웜 뉴트럴 + 4색 액센트 규율 유지.

## 3. 디자인 토큰
### 신규 — 다크 팔레트 (히어로 전용)
| 토큰 | 값 | 용도 |
|---|---|---|
| `--void` | `#06080f` | 히어로 배경 보이드 |
| `--arc` | `#2f6bff` | 레이저 아크 블루(주) |
| `--arc-hi` | `#4d8cff` | 아크 하이라이트 |
| `--arc-core` | `#eaf3ff` | 핫코어(화이트-블루) |
| `--glass-glow` | `rgba(47,107,255,.5)` | 유리 카드 글로우 |

### 유지 — 라이트 팔레트 (바디)
- `--paper #fbfaf7` · `--ink #17191c` · 4색 액센트(blue `#2563eb`/green/red/amber) 그대로.
- **연결 고리:** `--arc(#2f6bff)`가 기존 `--focus(#2563eb)`와 동일 계열 → 다크 블루가 라이트
  인터랙션 블루로 자연 연결. **새 색 안 늘림.**

### 시그니처 = "레이저 스캔라인"
- 얇은 전기블루 수평선(핫코어 + 소프트 블룸). 히어로 변환 메커니즘이자 **라이트 바디 섹션
  구분선으로 재사용** → 영상→UI 모티프 관통.
- "제품명 바꿔도 재사용 가능?" 테스트를 통과 못 하는(= 우리만의) 요소.

## 4. 아키텍처 / 컴포넌트
현 사이트는 Next.js 16(App Router), 애니메이션 라이브러리 0(전부 CSS + 바닐라 rAF/IO).
스크럽도 **바닐라 rAF**로 구현 — framer-motion 불필요. `lenis`(부드러운 스크롤)는 **선택**:
우선 없이 구현, 스텝감 있으면 데스크탑 한정 추가.

### 컴포넌트 분해
- **`LiquidVideoCanvas`** (client) — 고정 풀스크린 `<video>`(loop 아님, muted, playsInline,
  preload=auto, object-cover) + 스크롤→`currentTime` 시킹 로직. 단일 책임 = "스크롤 진행률을
  받아 프레임을 그린다."
  - 입력: `progress`(0~1). 출력: 없음(DOM 직접 조작).
  - **시킹 로직(핵심):** rAF 루프에서 목표 currentTime을 LERP(0.12)로 접근. 실제 seek는
    `if (!isSeeking && !video.seeking)` 가드 통과 시에만; 아니면 `nextSeekTime`에 큐잉.
    `seeked` 이벤트에서 큐 소진. → 검은 프레임/스터터 방지.
  - 엔트런스: `readyState>=3`에서 scale 1.12→1.0, opacity 0→1 (1.4s ease-out), 3.5s 안전
    타임아웃 폴백.
  - 에러: 연속 에러 카운트(max 3 재시도).
- **`ScrubHero`** (client) — 핀 고정 섹션 오케스트레이션. 세로로 긴 래퍼(높이 = 스크럽 거리,
  기본 `300vh` 튜닝값) 안에 `sticky top:0 h:100svh` 스테이지. 스크롤 진행률 계산 →
  `LiquidVideoCanvas`에 전달 + 카피 비트 + 다크→라이트 워시 + 대시보드 재조립 트리거.
- **`HeroDashboard`** (컴포넌트 추출) — 기존 페이지 인라인 대시보드(주인공 카드 카운트업 +
  보조 카드 4)를 컴포넌트로 추출. 스크럽 페이오프에서 렌더 + 모바일 폴백에서 직접 렌더(재사용).
- **CTA/Nav 글로우 버튼** — 대부분 CSS(회전 conic-gradient 테두리). 신규 컴포넌트 불필요.
- **스캔라인 구분선** — CSS 유틸 클래스(`.scanline`) + 필요시 소형 마크업.

### z-index 층위 (엄수)
`video z-1` < `스크럽 콘텐츠 z-10` < `하단 progressive blur z-30` < `nav z-50`.
- **다크 오버레이 금지**: 영상 위 반투명 검정·틴트·그라디언트 오버레이 절대 금지(영상 자연 밝기).
  래퍼 자체 `bg:var(--void)`만(영상 로드 전 노출용).

### 다크 → 라이트 해소
- 스크럽 `~85%`부터 페이퍼-화이트 워시가 페이드인(스캔라인 잔광과 함께) → 라이트 바디로 이음.
- 대시보드 재조립은 `~90–100%` 구간에서 유리 카드가 스태거로 조립.

### 모바일 / reduced-motion 폴백
- **모바일(터치 or `<768`):** 스크럽 미사용(iOS Safari `currentTime` 시킹 불안정). 히어로 =
  일반 높이 섹션, 선형 영상 **1회 autoplay(muted, playsInline) 후 정지** + `HeroDashboard` 바로
  노출.
- **`prefers-reduced-motion`:** 스크럽·엔트런스 스킵. 포스터(확정 스틸) + `HeroDashboard`
  즉시 표시. (현 사이트 reduced-motion 처리 관례 준수.)

## 5. 스크럽 카피 비트 (스크롤 마일스톤)
| 진행률 | 영상 상태 | 카피 |
|---|---|---|
| 0% | 종이 EMR 더미(정지) | "차트는 길다." |
| ~50% | 레이저 스캔 중 | "봐야 할 것만, 남긴다." |
| 100% | 유리 대시보드 재조립 | 기존 "12문장" 카운트업 + 보조카드 4 + **[시작하기] 글로우 버튼** |

- 카피 등장/퇴장 = 진행률 구간 기반 opacity/translate(현 `.hv`/`--hd` 패턴 재사용). 스크램블
  효과는 **비채택**(과한 스킨 — 우리 잉크-와이프 어휘 유지).

## 6. 라이트 바디 변경 (기존 유지 + 모티프 관통)
데모(S2)·25분과(S3)·CTA(S5)는 **웜 라이트 그대로**(임상 신뢰) — 단:
- 섹션 사이 구분선 = **레이저 스캔라인** 모티프(얇은 블루 글로우 라인).
- 데모의 "문장→원문 연결선"(`.ld-wire`)에 블루 아크 글로우 살짝.
- **Get Started 호버 = 회전 글로우 테두리**: 현 `.navpill.cta`(4색 흐르는 배경) 유지 +
  호버 시 **conic-gradient 광선이 테두리를 회전**(소프트 블룸). `mask`/`padding-box` 테크닉으로
  테두리만 광선(내부 배경 불침범). reduced-motion에서 회전 정지.
  - CTA 섹션 `.ld-cta` 버튼에도 동일 글로우 적용.

## 7. 딜리버러블 (사용자 → Claude)
- 최종 **선형 영상**(루프X): 종이더미 → 레이저 스캔 → 완전 유리(→ 우측 적재). Veo/Seedance에서
  생성.
- mp4 수령 후 Claude 담당: **스크럽용 인코딩**(키프레임 촘촘히 재인코딩 — 부드러운 시킹),
  webm/mp4 변환, 포스터(첫 프레임) 추출, `public/` 교체.
- **현 단계:** 확정 스틸을 포스터/플레이스홀더로 깔고 전체 골격부터 구축(영상 없이도 스크럽
  로직·레이아웃·라이트 바디 완성).

## 8. 비범위 (YAGNI)
- 스크램블 텍스트 효과(SynapseX 스킨) — 미채택.
- Space Mono 폰트 교체 — 미채택.
- framer-motion 도입 — 불필요(바닐라 rAF).
- `/app`(제품) 화면 개편 — 이번 범위 아님(랜딩만).
- 전체 다크화 — 기각됨(라이트 바디 유지).

## 9. 검증 / 테스트
- **결정론 게이트:** 구현 후 `node ~/.claude/tools/headless/sloplint.mjs <url>`(anti-slop 11규칙)
  → 기본값 수렴이면 재작업.
- **시각:** `/vcheck`(데스크탑+모바일 스샷, 가로 오버플로, 콘솔에러).
- **스크럽 수동 확인:** 스크롤 업/다운 시 검은 프레임·스터터 없음(`!seeking` 가드 검증), 모바일
  폴백 동작, reduced-motion 경로.
- **접근성:** 스크럽 콘텐츠는 스크롤 없이도 도달 가능해야(모바일/RM 폴백이 이를 보장).

## 10. 열린 항목
- 스크럽 래퍼 높이(`300vh`?) — 구현 중 체감으로 튜닝.
- `lenis` 도입 여부 — 바닐라로 먼저, 스텝감 있으면 추가.
- 대시보드 재조립 연출 세부(스태거 순서·유리 in 트랜지션) — 구현 중 시각 확정.
