## 0. 컨셉 — "종이 위의 시네마" (Light Cinematic Clinical)

시네마틱 = 다크가 업계 기본값. 우리는 **밝은 종이(진료기록지) 위에서 잉크가 움직이는 시네마**로 간다.
소재는 전부 제품 자신의 것: ECG 트레이스·비식별 차트 텍스트·분과 4색·mono 수치. 장식용 3D/글로우/파티클 금지.

- 시그니처 모티프: **ECG 잉크 라인** (자가 드로잉, 스크롤에 반응) + **차트 종이 대기** (비식별 텍스트 부유).
- 기존 임상 클래리티 토큰 계승: paper `#FBFAF7` · ink `#17191C` · IBM Plex Sans/Mono · 분과 4색. 랜딩은 이 토큰의 **볼륨업 버전**(디스플레이 타이포 스케일만 추가).

## 2. 랜딩 섹션 명세 (섹션당 카피 ≤2줄 + 시각 1개 — 초과 금지)

### S0. 히어로 — A+B 하이브리드 (100vh) — 2026-07-04 리서치 반영 확정

**리서치 근거 (2건 병렬 조사):**
- 어워드 조사: Stripe·Vercel·Linear급 히어로는 영상 파일이 아니라 **코드 생성 모션**(WebGL/CSS). 영상을 쓰는 곳(Runway·Notion·Webflow)은 전부 **제품 시연형**. 라이트 시네마틱 공식 = 여백+오버사이즈 타이포+절제된 슬로우 모션+무음 루프.
- 의료 조사: 임상 AI 경쟁군(Abridge·Nabla·Corti·Suki·DeepScribe·Ambience) **6/6 히어로 영상·실사 인물 없음**. 신뢰 문법 = "**motion that explains, not decorates**". 허용 모션 = 제품이 동작하는 걸 보여주는 것뿐.

**확정 콘티 — "차트가 요약으로 증류되는 6초" (제품 시연형 시네마 루프):**
1. 밝은 종이 위로 긴 비식별 차트 텍스트(mono)가 느리게 흐른다 (0–2s)
2. 핵심 문장 3곳이 잉크 하이라이트로 살아나고 (2–3.5s)
3. 우측으로 증류되어 요약 블록으로 정렬, 근거 연결선 1개가 그어진다 (3.5–5s)
4. ECG 잉크 라인이 하단을 가로지르며 씬 리셋 = 심리스 루프 (5–6s)
- 필름 트리트먼트: 종이 질감·미세 그레인·소프트 광 스윕·초미세 드리프트 (코드 티 제거)

**구현 형식 (B 레이어):** 어워드 실제 패턴대로 **라이브 코드 렌더**(캔버스) — 시각적으론 "영상 루프"로 읽히지만 파일이 아니라서: 용량 0·심리스 루프·LCP 즉시 페인트·리사이즈 선명. (녹화 webm은 폴백/OG용으로만 생성.)
**A 레이어 (스크롤 반응):** 같은 캔버스가 스크롤 진행률에 반응 — 스크롤 시작하면 증류 장면이 사용자 스크롤을 따라가고, ECG 라인이 아래로 꺾여 S1로 이어짐(섹션 연결자).
- 카피: `차트는 길다. 봐야 할 것은 짧다.` + 서브 1줄. CTA 버튼 1개 `차트 넣어보기 →`.
- 금지(리서치 교차검증): 실사 인물·청진기·흰가운 클리셰, bouncy 이징, 장식 전용 배경 모션, 범용 스톡.

## 3. 모션 규칙
- 라이브러리 무추가 — IntersectionObserver + rAF + CSS (기존 스택). 총 JS ≤ ~150줄.
- `prefers-reduced-motion` 또는 no-JS → 모든 씬 정적 완독 가능(영상은 poster로 대체).
- 60fps 원칙: transform/opacity만 애니메이트. 레이아웃 트리거 속성 금지.
- 모바일: 영상 대신 poster + 경량 캔버스(배터리·데이터 배려), 스크롤 씬은 유지.
